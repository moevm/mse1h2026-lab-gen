# Концепция ЛР6 - Связные списки

## 1. Формулировка задания

Необходимо реализовать набор функций для работы со связным списком целых чисел.

Структуры `ListStruct` и `ListNode` уже объявлены в проверяющей системе. Повторно объявлять их в решении не нужно.

Программа студента должна содержать только реализации требуемых функций. Функцию `main()` писать не нужно.

Каждый студент получает индивидуальный вариант, который определяется по `seed(студент, lab6)`.

Вариант задаёт:

- тип списка;
- набор обязательных стандартных функций;
- одну дополнительную функцию вставки;
- несколько дополнительных core-функций из общего пула;
- параметры для core-функций.

Цель лабораторной работы - научиться реализовывать базовые операции над разными видами связных списков и учитывать особенности конкретной структуры: обычного односвязного, циклического двунаправленного, развёрнутого и XOR-списка.

## 2. Флаги варианта

### `LIST_TYPE`

Определяет тип списка:

- `0` - односвязный список;
- `1` - циклический двунаправленный список;
- `2` - развёрнутый связный список;

Core-функции не включают стандартные функции списка. Стандартные функции всегда остаются обязательными и не участвуют в случайной выдаче core-функций.

### Параметры core-функций

Для некоторых core-функций используются параметры, вычисляемые из `seed(студент, lab6)`: `X`; `D`; `A`, `B`; `P`, `K`; `M`; `R`; `S`; `VALUE`.

## 3. Описание типов списков

## 3.1. Односвязный список

Список хранит значения типа `int`.

Односвязный список состоит из узлов, каждый из которых хранит значение и указатель на следующий элемент.

```c
struct ListNode {
    struct ListNode *next;  // указатель на следующий элемент
    int data;               // данные для хранения в узле
};

struct ListStruct {
    struct ListNode *head;  // указатель на голову
};
```

Особенности:

- `head` указывает на первый элемент списка;
- пустой список имеет `head == NULL`;
- последний элемент имеет `next == NULL`;
- движение возможно только вперёд;
- чтобы найти предыдущий узел, нужно начинать обход с головы;
- вставка в начало выполняется просто;
- вставка в конец требует обхода списка, если в структуре нет указателя на хвост.

## 3.2. Циклический двунаправленный список

Список хранит значения типа `int`.

Циклический двунаправленный список состоит из узлов, каждый из которых хранит значение, указатель на следующий элемент и указатель на предыдущий элемент.

```c
struct ListNode {
    struct ListNode *next;  // указатель на следующий элемент
    struct ListNode *prev;  // указатель на предыдущий элемент
    int data;               // данные для хранения в узле
};

struct ListStruct {
    struct ListNode *head;  // указатель на голову
};
```

Особенности:

- `head` указывает на первый элемент списка;
- пустой список имеет `head == NULL`;
- список является циклическим;
- у последнего элемента `next` указывает на `head`;
- у первого элемента `prev` указывает на последний элемент;
- у списка из одного элемента `node->next == node` и `node->prev == node`;
- при обходе нельзя искать конец по `NULL`;
- обход заканчивается, когда текущий указатель снова стал равен `head`;
- при вставке и удалении нужно корректно обновлять и `next`, и `prev`.

Типовой обход:

```c
if (list->head != NULL) {
    struct ListNode *current = list->head;
    do {
        // обработка current
        current = current->next;
    } while (current != list->head);
}
```

## 3.3. Развёрнутый связный список

Развёрнутый связный список - это структура данных, в которой каждый физический узел, то есть блок, содержит несколько логических элементов в массиве фиксированного размера.

Это уменьшает количество выделений памяти и снижает накладные расходы на хранение указателей.

Список хранит значения типа `int`.

```c
#define BLOCK_SIZE 4

struct ListNode {
    int data[BLOCK_SIZE];   // массив элементов
    int count;              // текущее количество элементов в блоке
    struct ListNode *next;  // указатель на следующий блок
};

struct ListStruct {
    struct ListNode *head;  // указатель на первый блок
};
```

### Что является узлом

В развёрнутом списке один `ListNode` - это не один элемент, а блок элементов.

Например, список из 9 чисел при `BLOCK_SIZE = 4` может выглядеть так:

```text
head
 |
 v
[3 8 1 6 | count = 4] -> [5 9 2 4 | count = 4] -> [7 _ _ _ | count = 1] -> NULL
```

Логические элементы списка:

```text
3, 8, 1, 6, 5, 9, 2, 4, 7
```

Физические узлы списка:

```text
block 0: 3, 8, 1, 6
block 1: 5, 9, 2, 4
block 2: 7
```

### Инварианты развёрнутого списка

При реализации нужно поддерживать следующие правила:

- пустой список имеет `head == NULL`;
- каждый существующий блок имеет `0 < count <= BLOCK_SIZE`;
- блок с `count == 0` не должен оставаться в списке;
- в последнем блоке может быть меньше `BLOCK_SIZE` элементов;
- элементы внутри блока находятся в `data[0] ... data[count - 1]`;
- элементы на позициях `count ... BLOCK_SIZE - 1` считаются неиспользуемыми;
- порядок логических элементов определяется порядком блоков и порядком элементов внутри блока.

### Добавление элемента в конец

Если список пуст, создаётся первый блок, а элемент записывается в `data[0]`.

Если последний блок заполнен не полностью, элемент добавляется в конец массива этого блока.

Если последний блок заполнен полностью, создаётся новый блок.

```text
До:
[1 2 3 _ | count = 3] -> NULL

После добавления 10:
[1 2 3 10 | count = 4] -> NULL
```

Если блок был заполнен:

```text
До:
[1 2 3 4 | count = 4] -> NULL

После добавления 10:
[1 2 3 4 | count = 4] -> [10 _ _ _ | count = 1] -> NULL
```

### Добавление элемента в начало

Если первый блок заполнен не полностью, элементы в нём сдвигаются вправо, а новый элемент записывается в `data[0]`.

Если первый блок заполнен, можно создать новый блок перед текущим `head`.

```text
До:
[2 3 4 _ | count = 3] -> NULL

После добавления 1:
[1 2 3 4 | count = 4] -> NULL
```

### Удаление логического элемента

При удалении элемента из блока элементы справа от него сдвигаются влево, а `count` уменьшается на 1.

Если после удаления `count == 0`, блок нужно удалить из списка.

```text
До удаления 8:
[3 8 1 6 | count = 4] -> [5 9 _ _ | count = 2]

После:
[3 1 6 _ | count = 3] -> [5 9 _ _ | count = 2]
```

### Доступ по индексу

Индекс считается по логическим элементам, а не по блокам.

```text
[3 8 1 6 | count = 4] -> [5 9 2 4 | count = 4] -> [7 _ _ _ | count = 1]

index: 0 1 2 3    4 5 6 7    8
data:  3 8 1 6    5 9 2 4    7
```

Чтобы получить элемент с индексом `6`, нужно:

1. пройти первый блок и пропустить 4 элемента;
2. перейти во второй блок;
3. вычислить локальный индекс `6 - 4 = 2`;
4. получить значение `data[2]` во втором блоке.

### Ограничение интерфейса

Для развёрнутого списка `ListNode` является блоком, а не отдельным логическим элементом.

Поэтому функции, принимающие указатель на конкретный логический элемент, для развёрнутого списка не выдаются.

Для развёрнутого списка операции над конкретным элементом должны выполняться через логический индекс.

## 3.4. XOR-список

XOR-список - это разновидность двусвязного списка, в котором вместо двух отдельных указателей `prev` и `next` используется один указатель `link`.

В поле `link` хранится результат побитового XOR адресов соседних узлов:

```text
link = prev XOR next
```

Список хранит значения типа `int`.

```c
struct ListNode {
    struct ListNode *link;  // XOR(prev, next)
    int data;               // данные
};

struct ListStruct {
    struct ListNode *head;  // указатель на первый элемент
};
```

Для выполнения XOR над адресами нужно использовать целочисленный тип `uintptr_t` из `<stdint.h>`.

```c
#include <stdint.h>

static struct ListNode *xor_nodes(struct ListNode *a, struct ListNode *b) {
    return (struct ListNode *)((uintptr_t)a ^ (uintptr_t)b);
}
```

### Как вычисляется следующий элемент

Чтобы получить следующий элемент при обходе, необходимо знать адрес предыдущего узла.

```text
next = prev XOR current->link
```

В коде:

```c
struct ListNode *next = xor_nodes(prev, current->link);
```

### Обход вперёд

```c
struct ListNode *prev = NULL;
struct ListNode *current = list->head;

while (current != NULL) {
    // обработка current

    struct ListNode *next = xor_nodes(prev, current->link);
    prev = current;
    current = next;
}
```

Для первого узла `prev == NULL`, поэтому:

```text
next = NULL XOR head->link
```

Если первый узел связан со вторым, результатом будет указатель на второй узел.

### Вставка в начало

При вставке нового узла в начало:

1. создаётся `new_node`;
2. старый `head` сохраняется как `old_head`;
3. `new_node->link = XOR(NULL, old_head)`;
4. если `old_head != NULL`, у старой головы нужно заменить соседа `NULL` на `new_node`;
5. `list->head = new_node`.

Если до вставки:

```text
old_head->link = XOR(NULL, second)
```

После вставки должно быть:

```text
old_head->link = XOR(new_node, second)
```

### Удаление узла

Для удаления текущего узла `current` нужно знать его соседей:

- `prev`;
- `next`.

После удаления:

- у `prev` нужно заменить соседа `current` на `next`;
- у `next` нужно заменить соседа `current` на `prev`;
- память под `current` нужно освободить.

Если до удаления:

```text
prev->link = XOR(prev_prev, current)
current->link = XOR(prev, next)
next->link = XOR(current, next_next)
```

То после удаления:

```text
prev->link = XOR(prev_prev, next)
next->link = XOR(prev, next_next)
```

Для этого предварительно вычисляются:

```text
prev_prev = XOR(prev->link, current)
next_next = XOR(next->link, current)
```

### Ограничения XOR-списка

- для движения вперёд нужно помнить предыдущий узел;
- для движения назад нужно помнить следующий узел;
- нельзя получить `next`, зная только `current`;
- нельзя получить `prev`, зная только `current`;
- произвольный доступ невозможен без последовательного обхода;
- любое неверное обновление XOR-связей разрушает список;
- отладка сложнее, чем у обычного списка;
- для преобразования указателей нужно использовать `uintptr_t`.

## 4. Обязательные стандартные функции

Все студенты реализуют следующие функции.

Эти функции не входят в пул core-функций и не выбираются случайно. Они всегда обязательны.

```c
struct ListStruct *list_init(void);
void list_destroy(struct ListStruct *list);
void list_push_front(struct ListStruct *list, int data);
void list_push_back(struct ListStruct *list, int data);
void list_pop(struct ListStruct *list, struct ListNode *node_to_remove);
int list_count(struct ListStruct *list);
struct ListNode *list_get_head(struct ListStruct *list);
struct ListNode *list_get_tail(struct ListStruct *list);
struct ListNode *list_get(struct ListStruct *list, int index);
bool list_is_empty(struct ListStruct *list);
void list_sort(struct ListStruct *list);
```

Описание стандартных функций:

| Функция | Назначение |
|---|---|
| `list_init()` | создать и вернуть пустой список |
| `list_destroy(list)` | удалить все элементы списка и освободить выделенную память |
| `list_push_front(list, data)` | добавить новый элемент в начало списка |
| `list_push_back(list, data)` | добавить новый элемент в конец списка |
| `list_pop(list, node_to_remove)` | удалить заданный элемент из списка |
| `list_count(list)` | вернуть количество логических элементов списка |
| `list_get_head(list)` | вернуть указатель на первый физический узел списка |
| `list_get_tail(list)` | вернуть указатель на последний физический узел списка |
| `list_get(list, index)` | вернуть указатель на элемент с заданным индексом |
| `list_is_empty(list)` | проверить, пуст ли список |
| `list_sort(list)` | отсортировать список по возрастанию значения `data` |

Для обычного, циклического двунаправленного и XOR-списка один физический узел соответствует одному логическому элементу.

Для развёрнутого списка один физический узел является блоком, поэтому при проверке функций нужно учитывать, что количество элементов считается как сумма `count` по всем блокам.

## 5. Дополнительная функция вставки

В зависимости от значения `INSERT_MODE` студент реализует одну из функций.

### `INSERT_MODE = 0`

```c
void list_insert_before(struct ListStruct *list, struct ListNode *node, int data);
```

Вставить новый элемент перед узлом `node`.

Если `node == NULL`, поведение фиксируется проверяющей системой. Рекомендуемый вариант - вставить элемент в конец списка.

### `INSERT_MODE = 1`

```c
void list_insert_after(struct ListStruct *list, struct ListNode *node, int data);
```

Вставить новый элемент после узла `node`.

Если `node == NULL`, поведение фиксируется проверяющей системой. Рекомендуемый вариант - вставить элемент в начало списка.

### Особенность для развёрнутого списка

Для развёрнутого списка указатель `struct ListNode *node` указывает на блок, а не на отдельный логический элемент.

Поэтому для `LIST_TYPE = 2` вместо `INSERT_MODE` можно использовать индексную вставку:

```c
void list_insert_at(struct ListStruct *list, int index, int data);
```

Вставка выполняется по логическому индексу.

Если `index <= 0`, элемент вставляется в начало.

Если `index >= list_count(list)`, элемент вставляется в конец.

```text
a <= data <= b
```

---

```text
P, P + K, P + 2K, ...
```

Параметры: `P`, `K`, где `K >= 1`.

---

---

---

### Код 10 - Проверка наличия элемента в диапазоне `[A; B]`

```c
bool list_has_range_value(struct ListStruct *list, int a, int b);
```

Вернуть `true`, если в списке есть хотя бы один логический элемент, удовлетворяющий условию:

```text
a <= data <= b
```

Иначе вернуть `false`.

Параметры: `A`, `B`, где `A <= B`.

---

### Код 11 - Удаление элементов вне диапазона `[A; B]`

```c
void list_remove_out_of_range(struct ListStruct *list, int a, int b);
```

Удалить все логические элементы, которые не попадают в диапазон `[a; b]`.

Остальные элементы должны сохранить относительный порядок.

Параметры: `A`, `B`, где `A <= B`.

---

### Код 12 - Сумма элементов в диапазоне индексов `[P; R]`

```c
int list_sum_index_range(struct ListStruct *list, int p, int r);
```

Посчитать сумму логических элементов с индексами от `p` до `r` включительно.

Если `p < 0`, считать с индекса `0`.

Если `r` выходит за пределы списка, считать до последнего элемента.

Если после нормализации диапазон пустой, вернуть `0`.

Параметры: `P`, `R`, где `R >= P`.

---

### Код 13 - Умножение элементов на позициях с шагом `K`

```c
void list_multiply_by_step(struct ListStruct *list, int p, int k, int m);
```

Умножить на `m` значения логических элементов с индексами:

```text
P, P + K, P + 2K, ...
```

Если `P` выходит за пределы списка, список не изменять.

Параметры: `P`, `K`, `M`, где `K >= 1`.

---

### Код 14 - Проверка чередования относительно `X`

```c
bool list_is_alternating_by_x(struct ListStruct *list, int x);
```

Проверить, чередуются ли элементы списка по отношению к `x`:

- один элемент меньше `x`;
- следующий больше либо равен `x`;
- затем снова меньше `x`;
- и так далее.

Допускается и обратный порядок начала: сначала элемент `>= x`, затем элемент `< x`.

Пустой список и список из одного элемента считаются корректными.

Параметр: `X`.

---

### Код 15 - Ограничение значений диапазоном `[A; B]`

```c
void list_clamp(struct ListStruct *list, int a, int b);
```

Изменить значения элементов списка так, чтобы все они попали в диапазон `[a; b]`:

- если `data < a`, заменить на `a`;
- если `data > b`, заменить на `b`;
- если `a <= data <= b`, оставить без изменений.

Параметры: `A`, `B`, где `A <= B`.

---

### Код 16 - Вставка `VALUE` после каждого элемента, кратного `D`

```c
void list_insert_value_after_divisible(struct ListStruct *list, int d, int value);
```

После каждого логического элемента, который делится на `d` без остатка, вставить новый элемент со значением `value`.

Вставленные элементы повторно не обрабатываются.

Параметры: `D`, `VALUE`, где `D != 0`.

Для развёрнутого списка вставка выполняется по логическим позициям внутри блоков.

---

### Код 17 - Удаление подряд идущих повторов `X`

```c
void list_compress_x_runs(struct ListStruct *list, int x);
```

Если в списке есть подряд идущие элементы, равные `x`, оставить в каждой такой группе только один элемент.

Пример при `x = 5`:

```text
1 5 5 5 2 5 5 3 -> 1 5 2 5 3
```

Параметр: `X`.

---

### Код 18 - Подсчёт пар соседей с суммой `S`

```c
int list_count_neighbor_pairs_sum(struct ListStruct *list, int s);
```

Посчитать количество пар соседних логических элементов, сумма которых равна `s`.

Пара считается по позициям `(0, 1)`, `(1, 2)`, `(2, 3)` и так далее.

Параметр: `S`.

Для циклического двунаправленного списка пара «последний + первый» не учитывается, если проверяющая система явно не задаёт обратное.

---

### Код 19 - Проверка, что все элементы с шагом `K` больше `X`

```c
bool list_all_by_step_greater(struct ListStruct *list, int p, int k, int x);
```

Проверить, что все логические элементы с индексами:

```text
P, P + K, P + 2K, ...
```

имеют значение больше `x`.

Если `P` выходит за пределы списка, вернуть `false`.

Параметры: `P`, `K`, `X`, где `K >= 1`.

## 8. Ограничения выдачи core-функций

Некоторые функции нельзя или нежелательно выдавать для отдельных типов списков.

| Функция | Ограничение |
|---|---|
| `list_find_first_greater` | для развёрнутого списка возвращает блок, а не конкретный элемент |
| `list_insert_value_after_divisible` | для развёрнутого списка требует вставки по логическому индексу внутри блоков |

Если требуется избежать неоднозначности, проверяющая система может не выдавать функции, возвращающие `struct ListNode *`, для `LIST_TYPE = 2`.
