# Концепция ЛР4

## Формулировка задания

Напишите программу на языке Си, которая читает строку элементов, обрабатывает её и выводит результат, используя функции стандартной библиотеки.

На вход программе подаются:

- строка с элементами;
- строка-запрос.

Элементы в первой строке разделены символами из набора разделителей, заданного правилом варианта.

Далее используются следующие термины:

- исходные элементы: элементы, полученные после разбиения первой строки по разделителям;
- пустой элемент: элемент нулевой длины, который может возникать между двумя соседними разделителями;
- выбранные элементы: элементы, которые программа оставила после проверки по правилу `select_rule`;
- отсортированные элементы: выбранные элементы после сортировки по правилу `sort_rule`;
- строка запроса: вторая входная строка, наличие которой среди отсортированных элементов определяется через `bsearch`;
- строка `summary`: строка, в которой выводится дополнительная характеристика по правилу `summary_rule`.

Программа должна:

- считать строку с элементами и строку-запрос;
- разбить первую строку на элементы;
- обработать пустые элементы по правилу `allow_empty_elements`;
- для каждого элемента проверить, подходит ли он под правило `select_rule`;
- отсортировать все выбранные элементы;
- обработать одинаковые элементы по правилу `allow_duplicates`;
- определить, присутствует ли строка-запрос среди выбранных элементов;
- вывести отсортированные элементы;
- после них вывести результат точного поиска;
- после этого вывести строку `summary`.

Общие требования для всех вариантов:

- чтение строк выполняется через `fgets`;
- разбиение строки на элементы выполняется средствами `string.h`;
- анализ элементов выполняется с помощью функций из `string.h` и `ctype.h`;
- сортировка выполняется через `qsort`;
- точный поиск выполняется через `bsearch`;
- если после отбора не осталось ни одного элемента, в первой строке выводится `empty`;
- результат программы должен быть однозначным и воспроизводимым для одного и того же входа.

В концепции задания фиксируются:

- общий набор core функций, одинаковый для всех студентов;
- вариативность:
  - `delimiter_rule`
  - `allow_empty_elements`
  - `allow_duplicates`
  - `select_rule`
  - `sort_rule`
  - `summary_rule`

### Core функции

Это общий набор действий, который одинаков для всех вариантов.

#### Чтение строк
Считать исходную строку элементов и строку-запрос.

#### Разбор строки на элементы
Разбить исходную строку на элементы средствами `string.h`, используя разделители, заданные правилом `delimiter_rule`.  
Для поиска ближайшего разделителя должны использоваться `strcspn` или `strpbrk`.

#### Обработка пустых элементов
Сохранять или отбрасывать пустые элементы в зависимости от правила `allow_empty_elements`.

#### Анализ элемента
Определять свойства элемента, которые нужны для правила `select_rule`.

#### Сортировка и обработка повторов
Сортировать выбранные элементы с помощью `qsort` и сохранять или удалять повторы в зависимости от правила `allow_duplicates`.  
При удалении повторов должно поддерживаться корректное уплотнение массива элементов.

#### Точный поиск
Определять наличие строки-запроса среди выбранных элементов с помощью `bsearch`.

#### Сборка итогового результата
Формировать строку отсортированных элементов, строку результата поиска и строку `summary`.

### Требования к `string.h`

Расширенная работа с `string.h` является частью концепции ЛР4.

В рамках этой концепции должны использоваться или явно поддерживаться следующие функции `string.h`:

- `strlen` - для работы с длиной элемента;
- `strcmp` - для лексикографического сравнения элементов;
- `strncmp` - для проверки префикса;
- `strstr` - для поиска подстроки;
- `strcspn` или `strpbrk` - для поиска ближайшего разделителя;
- `memcpy`;
- `memmove`.

Одна из целей ЛР4 - чтобы студент различал случаи применения `memcpy` и `memmove`:

- `memcpy` используется при копировании в непересекающиеся области памяти;
- `memmove` используется при копировании в потенциально пересекающиеся области памяти.

### Вариативные функции

У каждого студента есть шесть правил варианта:

- `delimiter_rule` - определяет, какие символы используются как разделители;
- `allow_empty_elements` - определяет, допускаются ли пустые элементы;
- `allow_duplicates` - определяет, сохраняются ли одинаковые элементы;
- `select_rule` - определяет, какие элементы остаются после отбора;
- `sort_rule` - определяет порядок сортировки выбранных элементов;
- `summary_rule` - определяет, какая дополнительная характеристика должна быть выведена.

Как именно работают эти правила, зависит от варианта студента.

### Порядок обработки

Обработка строки идёт по схеме:

1. считать строку элементов и строку-запрос;
2. разбить строку элементов по правилу `delimiter_rule`;
3. обработать пустые элементы по правилу `allow_empty_elements`;
4. для каждого полученного элемента применить `select_rule`;
5. отсортировать выбранные элементы по правилу `sort_rule`;
6. обработать повторы по правилу `allow_duplicates`;
7. определить наличие строки-запроса среди выбранных элементов через `bsearch`;
8. вывести отсортированные элементы;
9. вывести результат точного поиска;
10. вывести строку `summary`.

### Правила функции `delimiter_rule`

Студенту назначается одно правило набора разделителей.

#### Набор разделителей
Разделители элементов задаются непустым подмножеством из множества `{ ' ', '\t', ',', ';' }`.  

### Правила функции `allow_empty_elements`

Студенту назначается одно правило обработки пустых элементов.

#### Пустые элементы запрещены
Если между двумя соседними разделителями образуется пустой элемент, он не включается в дальнейшую обработку.

#### Пустые элементы разрешены
Если между двумя соседними разделителями образуется пустой элемент, он считается корректным элементом и участвует в дальнейшей обработке.

### Правила функции `allow_duplicates`

Студенту назначается одно правило обработки одинаковых элементов.

#### Повторы сохраняются
Одинаковые элементы после сортировки сохраняются в результирующем наборе.

#### Повторы удаляются
После сортировки одинаковые элементы схлопываются в один экземпляр.

### Правила функции `select_rule`

Студенту назначается одно правило отбора.

#### Отбор числовых элементов
Выбрать только те элементы, каждый символ которых является цифрой.  
Для проверки используется `isdigit`.

#### Отбор буквенных элементов
Выбрать только те элементы, каждый символ которых является буквой.  
Для проверки используется `isalpha`.

#### Отбор элементов в нижнем регистре
Выбрать только те буквенные элементы, каждый символ которых находится в нижнем регистре.  
Для проверки используется `islower`.

#### Отбор элементов в верхнем регистре
Выбрать только те буквенные элементы, каждый символ которых находится в верхнем регистре.  
Для проверки используется `isupper`.

#### Отбор элементов по префиксу
Выбрать только те элементы, которые начинаются с заданного префикса `P`.  
Для проверки используется `strncmp`.

#### Отбор элементов по подстроке
Выбрать только те элементы, которые содержат заданную подстроку `S`.  
Для проверки используется `strstr`.

### Правила функции `sort_rule`

Студенту назначается одно правило сортировки.

#### Лексикографическая сортировка по возрастанию
Отсортировать выбранные элементы лексикографически по возрастанию.

#### Лексикографическая сортировка по убыванию
Отсортировать выбранные элементы лексикографически по убыванию.

### Правила функции `summary_rule`

После сортировки выбранных элементов программа должна вывести одну дополнительную характеристику.

#### Количество выбранных элементов
Вывести количество элементов после отбора.

#### Первый и последний элемент после сортировки
Вывести первый и последний элементы отсортированного набора.

#### Длина самого длинного элемента
Вывести длину самого длинного выбранного элемента.

#### Длина самого короткого элемента
Вывести длину самого короткого выбранного элемента.

## Параметризация

### 1) Параметры, завязанные на хэш студента

Из `seed(студент, lab4)` детерминированно вычисляются:

- правило разделителей:
  - непустое подмножество из множества `{ ' ', '\t', ',', ';' }`;
- флаг обработки пустых элементов:
  - пустые элементы запрещены;
  - пустые элементы разрешены;
- флаг обработки одинаковых элементов:
  - повторы сохраняются;
  - повторы удаляются;
- правило отбора элементов:
  - только числовые элементы;
  - только буквенные элементы;
  - только элементы в нижнем регистре;
  - только элементы в верхнем регистре;
  - элементы с префиксом `P`;
  - элементы, содержащие подстроку `S`;
- правило сортировки:
  - лексикографически по возрастанию;
  - лексикографически по убыванию;
- правило дополнительной характеристики:
  - количество выбранных элементов;
  - первый и последний элемент после сортировки;
  - длина самого длинного элемента;
  - длина самого короткого элемента.

Студенту в тексте варианта явно показывается:

- какие символы считаются разделителями;
- разрешены ли пустые элементы;
- сохраняются ли одинаковые элементы;
- какие элементы должны быть отобраны;
- как должны быть отсортированы выбранные элементы;
- какая дополнительная характеристика должна быть выведена.

### 2) Общие параметры задания

Эти параметры задаются отдельно:

- `LineMax` — максимальная длина исходной строки;
- `ElementMax` — максимальное количество элементов;
- `WordMax` — максимальная длина одного элемента.

## Формат вывода

Чтобы проверка оставалась однозначной, результат программы предлагается зафиксировать так:

1. в первой строке вывести выбранные и отсортированные элементы через пробел;
2. во второй строке вывести результат точного поиска строки-запроса среди элементов:
   - `exists`
   - `doesn't exist`
3. в третьей строке вывести дополнительную характеристику.

Если после отбора элементов не осталось, в первой строке выводится:
- `empty`

Форма вывода для дополнительной характеристики:

- количество элементов:
  - `summary: count=...`
- первый и последний элемент:
  - `summary: first=... last=...`
- длина самого длинного элемента:
  - `summary: maxlen=...`
- длина самого короткого элемента:
  - `summary: minlen=...`

## Пример варианта

Пример варианта студента:

- разделители: пробел, табуляция и запятая;
- `allow_empty_elements`: пустые элементы запрещены;
- `allow_duplicates`: повторы удаляются;
- `select_rule`: элементы, содержащие подстроку `te`;
- сортировка: лексикографически по возрастанию;
- `summary_rule`: первый и последний элемент после сортировки.

Тогда программа должна:

- считать исходную строку и строку-запрос;
- разбить исходную строку на элементы по пробелам, символам табуляции и запятым;
- не включать пустые элементы в дальнейшую обработку;
- оставить только те элементы, которые содержат подстроку `te`;
- отсортировать их по возрастанию;
- удалить одинаковые элементы;
- определить, присутствует ли строка-запрос среди выбранных элементов через `bsearch`;
- вывести первый и последний элемент после сортировки.

Пример ввода:

```text
abc,	test,,beta test meter
test
```

Пример вывода:

```text
meter test
exists
summary: first=meter last=test
```
