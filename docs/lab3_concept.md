# Концепция ЛР3

## Формулировка задания

Напишите программу на языке Си, которая читает текст, обрабатывает его и выводит результат.

На вход программе подаётся текст произвольной длины.  
Предложения в тексте заканчиваются одним из символов `.`, `;`, `?`, `!`.

Конец текста задаётся специальным маркером `###`.  
Маркер `###` не является предложением, не обрабатывается и в вывод не входит.

Далее используются следующие термины:

- выбранные предложения: предложения, которые программа оставила после проверки по правилу `select_rule`;
- преобразованные предложения: выбранные предложения после изменения по правилу `rewrite_rule`;
- ключевое слово: слово, которое программа выбирает из преобразованного предложения по правилу `keyword_rule`;
- строка `Key words`: строка, в которой все ключевые слова записаны через пробел в том же порядке, в котором выводятся соответствующие предложения.

Программа должна:

- считать текст и разделить его на предложения;
- для каждого предложения проверить, подходит ли оно под правило `select_rule`;
- для каждого подходящего предложения применить правило `rewrite_rule`;
- из каждого получившегося предложения выбрать одно ключевое слово по правилу `keyword_rule`;
- вывести все получившиеся предложения;
- после них вывести строку `Key words: ...`.

Общие требования для всех вариантов:

- пробелы и табуляции в начале предложения удаляются;
- маркер `###` не входит в обработку;
- каждое предложение результата выводится с новой строки;
- порядок выбранных предложений не меняется;
- если после обработки не осталось ни одного предложения, выводится `EMPTY`;
- если после обработки не осталось ни одного предложения, строка ключевых слов имеет вид `Key words: EMPTY`.

В концепции задания фиксируются:

- общий набор core функций, одинаковый для всех студентов;
- вариативность:
  - `select_rule`
  - `rewrite_rule`
  - `keyword_rule`

### Core функции

Это общий набор действий, который одинаков для всех вариантов.

#### Чтение текста
Считать входной текст до маркера `###`.

#### Разбор текста на предложения
Находить в тексте отдельные предложения.

#### Нормализация предложения
Удалять пробелы и табуляции в начале предложения.

#### Разбор предложения на слова
Разбивать предложение на слова.

#### Анализ предложения
Определять свойства предложения и его слов, которые нужны для `select_rule`, `rewrite_rule` и `keyword_rule`.

#### Сборка предложения и итогового результата
Формировать преобразованное предложение и строку `Key words`.

### Вариативные функции

У каждого студента есть три правила варианта:

- `select_rule` - определяет, будет ли предложение обрабатываться дальше;
- `rewrite_rule` - преобразует выбранное предложение;
- `keyword_rule` - выбирает ключевое слово из преобразованного предложения.

Как именно работают эти три правила, зависит от варианта студента.

### Порядок обработки

Обработка текста идёт по схеме:

1. считать весь текст;
2. выделить очередное предложение;
3. применить `select_rule`;
4. для выбранного предложения применить `rewrite_rule`;
5. выбрать ключевое слово по правилу `keyword_rule`;
6. вывести преобразованное предложение;
7. после завершения обработки вывести строку `Key words`.

### Правила функции `select_rule`

Студенту назначается одно правило отбора.

#### Отбор по количеству слов
Выбрать предложение, если число слов в нём удовлетворяет условию с порогом `W`.  
Параметры: знак сравнения и `W`.

#### Отбор по завершающему знаку
Выбрать предложение, только если оно оканчивается одним из символов заданного множества.  
Параметр: непустое подмножество из `{'.', ';', '?', '!'}`.

#### Отбор по длине слова
Выбрать предложение, только если в нём есть слово длины не меньше `L`.  
Параметр: `L`.

#### Отбор по числу цифр
Выбрать предложение, только если общее число цифр в предложении не меньше `D`.  
Параметр: `D`.

#### Отбор по позиции
Выбрать только предложения с чётными или только с нечётными номерами.  
Параметр: тип позиции.

### Правила функции `rewrite_rule`

Студенту назначается одно правило преобразования.

#### Разворот порядка слов
Поменять порядок слов в предложении на обратный, сохраняя завершающий знак предложения.

#### Циклический сдвиг слов
Циклически сдвинуть слова влево или вправо на `T` позиций.  
Параметры: направление, `T`.

#### Обмен первого и последнего слова
Если в предложении не меньше двух слов, поменять местами первое и последнее слово.

#### Удаление слов по позиции
Удалить слова с чётными или с нечётными номерами.  
Если после удаления слов предложение стало пустым, оно не выводится.  
Параметр: тип позиции.

#### Дублирование первого слова
Продублировать первое слово в конец предложения перед знаком препинания.

### Правила функции `keyword_rule`

Из каждого преобразованного и непустого предложения выбирается одно ключевое слово.

#### Первое самое длинное слово
Выбрать самое длинное слово; при равенстве взять первое.

#### Последнее самое короткое слово
Выбрать самое короткое слово; при равенстве взять последнее.

#### Слово с наибольшим числом гласных
Выбрать слово, содержащее максимальное число гласных; при равенстве взять первое.

#### Слово по позиции
Выбрать слово с номером `P`, если оно существует, иначе последнее слово предложения.  
Параметр: `P`.


## Параметризация

### 1) Параметры, завязанные на хэш студента

Из `seed(студент, lab3)` детерминированно вычисляются:

- правило функции `select_rule`;
- параметры `select_rule`;
- правило функции `rewrite_rule`;
- параметры `rewrite_rule`;
- правило функции `keyword_rule`;
- параметры `keyword_rule`;
- текстовое описание варианта в формулировке задания.

То есть по хэшу студента определяется:

- какое правило отбора он получает;
- какое правило преобразования он получает;
- какое правило выбора ключевого слова он получает;
- какие значения параметров используются в этих правилах.

### 2) Параметры ввода

Эти параметры одинаковы для всех вариантов и задаются отдельно:

- `TextMax` - максимальная длина входного текста;
- `SentenceMax` - максимальное число предложений;
- `WordMax` - максимальная длина одного слова;
- маркер конца текста `###`;
- множество допустимых завершающих символов предложения.

## Пример варианта

Пример варианта студента:

- `select_rule`: выбрать только предложения, в которых есть слово длины не меньше `8`;
- `rewrite_rule`: циклически сдвинуть слова вправо на `2` позиции;
- `keyword_rule`: выбрать первое самое длинное слово.

Тогда программа должна:

- читать весь текст до маркера `###`;
- брать только те предложения, где есть хотя бы одно слово длины `8` и больше;
- в каждом таком предложении циклически переставлять слова вправо на `2`;
- из каждого преобразованного предложения брать самое длинное слово;
- выводить все получившиеся предложения;
- после них выводить строку `Key words` из выбранных ключевых слов.

Пример ввода:

```text
Small fox runs. Brilliant minds grow fast! Tiny digits stay? Northern mountains shimmer brightly. ###
```

Пример вывода:

```text
grow fast Brilliant minds!
shimmer brightly Northern mountains.
Key words: Brilliant mountains
```
