# Концепция ЛР1

## Формулировка задания

Напишите программу на языке Си, выделив каждую подзадачу в отдельную функцию.

На вход подаётся массив целых чисел.  
Размер массива не больше Nmax, числа разделены разделителем SEP, строка заканчивается символом перевода строки.

Программа должна последовательно вычислить все подзадачи, назначенные конкретному студенту, и вывести результаты в заданном порядке, каждый результат с новой строки.  
Ошибкой считается дублирование кода.

## 6 подзадач

### Первый элемент, кратный M
Вывести индекс первого элемента массива, который делится на M без остатка.  
Если нет — -1.  
Параметр: M.

### Последний элемент по модулю больше T
Вывести индекс последнего элемента массива, для которого |a[i]| > T.  
Если нет — -1.  
Параметр: T.

### Количество элементов в диапазоне [A; B]
Вывести количество элементов, удовлетворяющих A ≤ a[i] ≤ B.  
Параметры: A, B (A ≤ B).

### Сумма модулей на позициях с шагом K, начиная с P
Посчитать сумму |a[i]| по индексам P, P+K, P+2K, ... пока < n.  
Если P ≥ n — 0.  
Параметры: P, K (K ≥ 1).

### Сумма элементов, делящихся на D
Посчитать сумму всех элементов, которые делятся на D без остатка.  
Если нет — 0.  
Параметр: D.

### Количество элементов, которые по модулю меньше L
Посчитать количество элементов, для которых |a[i]| < L.  
Параметр: L.

## Параметризация

### 1) Параметры, завязанные на хэш студента

Из seed(студент, lab1) детерминированно вычисляются:

- Список подзадач: из 6 выбирается K разных подзадач (без повторов) и фиксируется их порядок вывода.
- Числовые параметры подзадач: M, T, A, B, P, K_step, D, L (только для тех подзадач, которые попали студенту).

Студенту в тексте варианта явно показывается, какие подзадачи ему назначены и какие у них значения параметров.

### 2) Параметры ввода, не завязанные на seed

Эти параметры задаются отдельно:

- Nmax — максимальный размер массива;
- SEP — формат разделения чисел.

### 3) Количество подзадач отдельный параметр

K (сколько подзадач получит студент) задаётся как отдельный параметр задания, и дальше:

- из 6 выбираются ровно K подзадач,
- программа выводит ровно K строк-результатов.
