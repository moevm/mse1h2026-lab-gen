# Концепция ЛР2

## Формулировка задания

Напишите многофайловый проект на языке Си, разбив решение на модули и собрав его через `Makefile`.

На вход подаётся массив целых чисел.  
Размер массива не больше `Nmax`, числа разделены пробелами, строка заканчивается символом перевода строки.

Программа должна считать массив, последовательно применить к нему все шаги конвейера, назначенные конкретному студенту, и вывести массив после каждого шага в заданном порядке.  
Состояние массива после последнего шага считается итоговым результатом программы.  
Ошибкой считается дублирование кода.

В проекте должны быть выделены:

- модуль `data_io` для ввода и вывода массива;
- набор `core`-функций, выполняющих базовые преобразования массива;
- набор `step`-функций, каждая из которых вызывает одну или несколько `core`-функций;
- главный файл, который вызывает `input`, затем `step`-функции по порядку и `output` после каждого шага.

В MVP `core`-функции работают с массивом на месте, не меняют его длину и имеют единообразный интерфейс вида `void func(int arr[], int size)`.

## Core-функции и step-функции

Вариант студента — это последовательность из `K` шагов конвейера.

Каждый шаг представляет собой отдельную `step`-функцию.  
Каждая `step`-функция должна быть реализована через вызов одной или нескольких `core`-функций.

### Core-функции

#### Разворот подмассива
Развернуть элементы массива с индексами от `A` до `B` включительно.  
Параметры: `A`, `B`, где `0 <= A <= B < n`.

#### Циклический сдвиг массива
Циклически сдвинуть массив влево или вправо на `T` позиций.  
Параметры: направление сдвига, `T`, где `T >= 0`.

#### Перестановка элементов в парах
Разбить массив на пары подряд идущих элементов и поменять элементы в каждой полной паре местами.  
Если длина массива нечётная, последний элемент остаётся на месте.

#### Перенос чётных элементов в начало
Перенести все чётные элементы в начало массива, а все нечётные — в конец.  
Требование устойчивости не предъявляется.

#### Преобразование в порядок «начало-конец»
Преобразовать массив в порядок `a0, a[n-1], a1, a[n-2], ...`.

### Step-функции

Каждая `step`-функция задаёт последовательное преобразование массива и может вызывать одну и ту же `core`-функцию несколько раз.  
Разные `step`-функции могут использовать общие `core`-функции.

Для MVP студенту назначаются `K` шагов, где `K` задаётся отдельно и обычно равно `2` или `3`.

## Параметризация

### 1) Параметры, завязанные на хэш студента

Из `seed(студент, lab2)` детерминированно вычисляются:

- список `core`-функций, доступных в варианте;
- число шагов `K` и порядок вызова `step`-функций;
- состав каждой `step`-функции, то есть какие `core`-функции и в каком порядке она вызывает;
- числовые параметры преобразований: `A`, `B`, `T` и другие параметры назначенных функций;
- имена `step`-функций и `core`-функций;
- имена `.c` и `.h` файлов модулей;
- имя главного файла;
- имя итогового исполняемого файла.

Студенту в тексте варианта явно показывается:

- какие `step`-функции он должен реализовать;
- какие преобразования выполняет каждый шаг;
- какие значения параметров используются;
- в каких файлах должны находиться реализации.

### 2) Параметры ввода, не завязанные на seed

Эти параметры задаются отдельно:

- `Nmax` — максимальный размер массива.

### 3) Количество шагов как отдельный параметр

`K` задаётся как отдельный параметр задания, и дальше:

- студент получает ровно `K` `step`-функций;
- программа выводит ровно `K` состояний массива: по одному после каждого шага конвейера.

## Формат сдачи

Студент отправляет решение одним текстом, разбитым на блоки файлов следующего вида:

```text
###main_ab12.c###
...

###data_io_q10.h###
...

###data_io_q10.c###
...

###core_r21.h###
...

###core_r21.c###
...

###step_k02.h###
...

###step_k02.c###
...

###Makefile###
...
```

Проверка должна:

- разобрать текст на отдельные файлы;
- сохранить их во временную директорию;
- выполнить `make`;
- запустить тесты;
- выполнить `make clean`.

Имена файлов, функций и исполняемого файла должны быть заданы в варианте студента и должны учитываться при проверке.
